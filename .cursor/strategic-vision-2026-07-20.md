## 0. エグゼクティブサマリー

TwoSidesは「分断より納得」を掲げる第三のメディアとしてスタートしたが、2026年7月18日までの状態は「便利だけど毎日開く理由に欠ける」サイトだった。本日7月20日の議論で以下を確定した：

- **コアバリュー**：「この争点、今どうなってる？」が一瞬でわかる
- **ターゲット**：政治に関心ある中立層（約1,200万人）に絞る。Z世代脳死スクロール層は対象外
- **プロダクト構造**：News（集客）→ YouTube討論（刺さり）→ Debate常時15〜20スレッド（本丸）→ スイングカード拡散（ループ）
- **収益**：広告（Free層）+ サブスクPlus/Pro（分析・詳細データ）
- **目標**：2026年10月末 月間1,000万PV → 2027年9月末 月間1億PV

## 1. 市場分析とターゲット

### 1.1 日本人の政治関心の3層

| 層 | 定義 | 人数 | 割合 |
|---|---|---|---|
| **A. 政治・社会に関心がある** | ニュースを自分から見る、選挙に行く | 約2,000〜2,500万人 | 16〜20% |
| **B. 関心はあるが受動的** | Yahooトップにあったら読む、Xで話題なら見る | 約3,000万人 | 24% |
| **C. ほぼ関心なし** | 政治の話はしない、選挙に行かないことも | 約7,000万人 | 56% |

### 1.2 層Aの内訳

| サブセグメント | 人数 | 特徴 | TwoSidesとの適合度 |
|---|---|---|---|
| **A1. 政治オタク / ウォッチャー** | 〜150万人 | 毎日複数メディアを巡回。Xで政治ポスト | ◎ 議論のエンジン。コメントを生む |
| **A2. ビジネスパーソン（情報収集必須層）** | 〜500万人 | 金融・コンサル・官公庁・メディアetc. 業務上政策動向を追う必要 | ◎ タイパ命。朝の情報収集に最適 |
| **A3. 意識高い系大学生・院生** | 〜200万人 | 政治に興味あり、議論したい | ○ クイズ＋スタンプ＋スプリットでハマる |
| **A4. 中立だけど気になる層【最重要ターゲット】** | **〜1,200万人** | 偏った情報は嫌。客観的に知りたい。でも自分から探すのは面倒 | **◎ ここが最大市場。誰も満足させてない** |
| **A5. 右派・左派活動層** | 各〜50万人 | 自分の主張を広めたい | △ コンテンツを生む。ただし毒性あり |

### 1.3 プライマリターゲット：A4（中立だけど気になる層）

**彼らの現状と不満：**

| 使ってるサービス | 不満 |
|---|---|
| Yahooニュースを見る | コメント欄が偏ってて気分悪い |
| Xで論争を見る | 声の大きい人しか見えない。「実際どうなの？」が分からない |
| まとめサイトを見る | 恣意的な切り抜き。信用できない |
| 何も見ない | 情報不足で不安 |

**彼らが本当に求めているもの：**

1. 「今、この争点はどうなってるのか」を正確に知りたい（偏ってない）
2. 「他の人はどう思ってるのか」を知りたい（声の大きい人じゃなく、普通の人の意見）
3. 「自分はどう思うのか」を確認したい（匿名投票なら安心）
4. 時間をかけたくない（仕事・子育ての合間）

### 1.4 TwoSidesが満たすもの

| 欲求 | TwoSidesの機能 |
|---|---|
| 偏ってない情報 | bothSides品質ゲート + 複数ソース横断 |
| 他の人の意見 | リアルタイム投票結果・スイング |
| 自分の立場確認 | 匿名3択投票（賛成/わからない/反対） |
| 時間をかけない | 30秒要約 + bullets |

### 3.3 層3: Debate（本丸）

**目的：** 「今どうなってる？」で一日何度も開かれる常設議論空間

**常時アクティブなスレッド数の計算：**

| ソース | 1日あたり新規 | 寿命 | 常時アクティブ数 |
|---|---|---|---|
| 現行Discover（News由来のDebate） | 5本 | 3〜7日 | 15〜20 |
| YouTube討論由来 | 5〜7本 | 3〜7日 | 20〜28 |
| 海外討論由来 | 2〜3本 | 3〜7日 | 8〜12 |
| AI検出バズり動画由来 | 2〜3本 | 5〜14日 | 10〜20 |

**理論上の常時アクティブ数：15〜25スレッド（最低でも10〜12は安定）**

**1スレッドの構造（再掲）：**

```
ページ上部：リアルタイム投票結果（3色バー + スイング）
　　　　　「中立層が賛成側に+2.3pt」のインジケータ
　　　　　（これがTOPにある。要約はその後）

中部：スプリットスレッド（賛成コラム / 中立コラム / 反対コラム）
　　　「コメントを書く」じゃなく「自分の陣営に追加する」感覚
　　　各コラムには最新の意見が1〜2件表示される

下部：30秒要約（lead + bullets）
　　　「議論だけ見たい人」はここを読まなくてもいい

最下部：この争点の詳細リンク（動画埋め込み / 関連News）
```

**Debateが「生きてる」を維持する仕組み：**

| 仕掛け | 効果 |
|---|---|
| 投票がリアルタイムで動く（SSE or 60秒polling） | 「今どうなった？」で何度も開く |
| スイングが変化する（中立の動き） | 朝・昼・夜で違う顔を見せる |
| コメントが積み上がる | 新しい意見が出るとまた開く |
| Xで誰かがスイングカードをシェアする | また流入が起きる |
| プッシュ通知（「新しい意見が15件」） | 投票した人だけに送る |

**常設Debate（超長期テーマ）：**

| テーマ | 理由 |
|---|---|
| 消費税（減税/増税） | 永遠の争点。政治が動くたびに活性化 |
| 憲法改正 | 議論が続く限り生きている |
| 皇室典範 | 愛子天皇問題で今一番ホット |
| 原発（再稼働/廃炉） | エネルギー政策の核心 |
| 外交（日米/日中/日韓） | 国際情勢で常に動きがある |

常設Debateは「今日も議論が続いてる」が視認できるように、トップに固定表示。

### 3.4 層4: スイングカードX拡散（ループ）

**目的：** 拡散ループを回して新規流入を自己増殖させる

**スイングカードのデザイン：**

```
┌────────────────────────────┐
│  TwoSides                    │
│                              │
│  消費税減税、与党内で調整中   │
│                              │
│  賛成 42% │ 中立 18% │ 反対 40│
│                  ↓           │
│        中立 +2.3pt → 賛成側  │
│                              │
│  n=1,247  直近6時間           │
│  議論を見る → twosides.jp/..  │
└────────────────────────────┘
```

**このカードが拡散される理由：**

- **データだから誰でもシェアできる**（煽りゼロ、中立）
- 「今の空気」を一発で伝える
- 政治ツイッタラーの恰好のネタになる
- 政治家のスタッフも共有する（自分のボスに関係する争点なら特に）
- メディア関係者が「今の世論」として引用する

**拡散ループの流れ：**

```
スイングカード画像 → Xに投稿 → バズる
                      → クリック → TwoSidesのDebateへ
                      → 投票する → スイングが更新される
                      → またカードが生成される → Xに…
```

### 3.6 層6: SEO巨大資産

**目的：** 検索からの安定的な流入基盤を築く。Yahooの20年の蓄積に対抗するには「数」が必要

#### 3.6.1 政治家ページ（約700ページ）

**対象：** 衆議院465 + 参議院248 + 主要政党10 = 約700ページ

**1ページの構成：**

```

   - 内訳：中立層の60%が支持に流れている

4. 説得力スコア（既存）
   - 争点横断のbridgingスコア

5. 争点での賛否履歴（既存）
   - この政治家が関わった争点一覧

#### 3.6.2 用語ページ（10,000ページ目標）

**発想：** 記事中のglossary（GPIF・乖離許容幅等）を、個別の用語解説ページとして自動生成する。

```
/glossary/GPIF

**自動生成フロー：**

```

```

**効果：**

| 指標 | 数値 |

| 1記事あたりの新規ページ数 | 3〜5ページ |
| 1日あたり（30記事） | 90〜150ページ |
| 100日後 | 9,000〜15,000ページ |
| SEO流入（6ヶ月目） | 月間200万〜500万PV |

**検索クエリ例：** 「GPIF とは」「乖離許容幅 意味」「日経平均 とは」

### 3.7 層7: スイングデータのメディア露出（爆発トリガー）

**目的：** テレビ・新聞でTwoSidesのデータが引用される瞬間、流入が指数関数的に跳ねる

**狙い方：**

| 対象 | アプローチ |
|---|---|
| **アベプラなどの番組プロデューサー** | 「番組で取り上げたテーマの、視聴者のリアルタイム投票データを提供できます」 |
| **政治部記者** | 「△△大臣の支持スイング、直感的に分かるグラフありますよ」 |
| **YouTuber（政治系）** | 「あなたの動画についての議論がTwoSidesで行われています。データ提供します」 |

## 9. 意思決定の判断基準

今後このドキュメントに沿って判断するときの基準：

1. **「今どうなってる？」に答えられているか**
   - すべての機能はこの一言に収束する。これに貢献しない機能は後回し

2. **中立層（A4）に刺さるか**
   - Z世代脳死層向けの施策（脳死スワイプ・連続ログインストリーク等）は優先度を下げる
   - 「時間をかけずに正確な空気を知りたい」に応えるものだけを作る

3. **拡散ループに入るか**
   - 閉じた体験で終わらせない。外に漏れ出る設計になっているか
   - 「これシェアしたい」と思える単位（画像・カード・データ）が含まれているか

4. **既存の7層のどれに貢献するか**
   - どの層を強化する施策か明確に説明できること
   - どれにも貢献しない施策は採用しない

5. **「対立の結果でなく説得の過程を商品化する」に反しないか**
   - 2026-07-18に確定した原則。誹謗中傷・対立煽り・空ドーパミンを報酬にしない
   - 理解が残ったかどうかを報酬の基準にする

6. **1日10分で完結するか**
   - ユーザーに負荷をかけすぎない。習慣化には低負荷が必須
   - 「読むのが面倒」じゃなく「気づいたら読んでた」を目指す

### 政治家ページの設計方針確定
「人格への支持/不支持」ではなく「行動・発言・国会での賛否」を争点連動で積み上げる設計にする。理由：政治家は公人であり、人格でなく行動・発言・国会活動に対して賛否されるべきだが、それを日々のニュースと一貫して追える場所が現状ない（好き嫌い.com型の人気投票との違いはここ）。**投票記録データはLLM生成でなく国会会議録検索システム(kokkai.ndl.go.jp)等の一次データから機械的に取り込む**（誤り時の実害が記事より大きいため）。プロフィール文のみAI生成可、投票記録は一次データリンク直結。政治家ページからも該当Debateへの導線（CTA）を必須にする。

3. **スイングカードX拡散（4番）は最も検証されていない不確実な変数**。§3.4の「拡散ループ」は仮説段階。ここが機能しないと新規流入の自己増殖が起きず、SEOだけに頼る成長曲線になる。P0で最優先級に位置づけ、早期に実データで検証すべき（4〜17の順序を入れ替えてでも早めるべき候補）。
4. **YouTube討論（1〜2番）はリテンションの「刺さり」担当であり、単体のPV貢献は小さい（§6で300万PV程度）が、Debate常連化への入口として機能する**。過大投資せず、5〜7本/日程度の規模を維持すれば十分（1日30〜35本の供給計画で確定済み）。
5. **監査体制（5番）は成長のボトルネックになりうる唯一の非機能要件**。政治家ページ・YouTube討論・バズり動画Debateが同時にスケールし始めると通報・炎上対応の負荷が跳ねる可能性があるため、急上昇検知の実装（5番）はP0の中でも早めに済ませておく。

**結論**：P0の中でも実行順は 5(監査インフラ)→8(PWA)→4(スイングカード)→3(政治家ページMVP)→1・2(YouTube系) を目安にする。監査インフラと最大レバー（PWA）を先に固め、不確実性の高いスイングカード拡散を早期検証し、SEO系は並行して仕込みを開始する、という順序が「行き当たりばったりでなく複利を効かせる」成長曲線に最も近い。
