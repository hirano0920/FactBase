# TwoSides 記事品質ルーブリック（v2 — 一般人クリック基準に修正）
**10軸 各1〜5点 = 50点満点 / 43点以上 = プロ品質(95点相当)**

---

## 軸1: リードのフック力（hook）— 5点
**「この記事を読みたい」と思わせる冒頭か**

| 点数 | 基準 |
|------|------|
| 5 | 冒頭の1文で「何が起きたか・なぜ争点か・自分に関係あるか」の3つ一気に伝わる。専門用語は即座に言い換え（「GPIF→年金を運用する機関」等）。固有名詞・数値・行為が最初に出るが、一般読者に意味が通じる形で出る |
| 4 | 1文目で争点の核心には触れているが、一般読者への波及が弱い or 専門用語をそのまま使っていて一部の読者しか理解できない |
| 3 | リードは正しいが教科書的。「〜を巡る議論」などの抽象切り出し。フック不足 |
| 2 | ニュース性が伝わらない。リードが背景説明で始まる／テンプレ的な書き出し |
| 1 | リードが無い／issueTitleのコピペ／「世の中では〜が問題になっています」等の中学生作文 |

**ジャッジポイント**: 政治に詳しくない友達に見せて「え、何それ？」と言われるか、「ああ、それ知ってる」と言われるか。

---

## 軸2: タイトルの一般人クリック率（title）— 5点
**issueTitle（リスト表示・スレッドヘッダ）として、専門知識がない一般ユーザーが「自分に関係ある」と思ってクリックしたくなるか**

| 点数 | 基準 |
|------|------|
| 5 | 専門用語をそのまま使わず、一般読者に意味が通る形に翻訳している。かつ「自分に関係ある」と思わせる生活フックがある。擬似例:「年金運用のGPIF、国内投資を拡大——私たちの年金への影響は？」 |
| 4 | 固有名詞はあるが、専門用語を説明なしで使っているため知識のある人にしか伝わらない。GPIFの例なら、フックはあるが「GPIF」を説明していない |
| 3 | 固有名詞がなく抽象的だが、代わりにわかりやすい生活フックがある（「年金の運用方法が変わる？私たちへの影響」等） |
| 2 | 固有名詞がなく、かつ抽象的。読者が「よくわからないからスルー」する。例:「被災対応は適切だと思いますか？」（どの災害？何が？） |
| 1 | タイトルが政治家名言／キャッチコピー風／「驚きの事実」等のクリックベイト／または「論文盗用で学位剥奪は妥当ですか？」のように固有名詞ゼロ・フックゼロ |

**ジャッジポイント**: **「お母さんでも意味がわかるか？」** 専門知識が無い人に「このタイトル、何の話？」と聞いたとき即答できるか。

## 軸3: 両論のフェアネス（bothSides）— 5点（最重要・2倍加重）
**両陣営の主張が「具体的・同等・フェア」に書かれているか**

| 点数 | 基準 |
|------|------|
| 5 | 両側とも3〜4項目・同じ分量・同じ根拠密度。各項目に媒体/発言者帰属あり。両側の「最強の理由」が揃っている |
| 4 | 両側とも十分な具体性。ただし一方がやや薄い／帰属が2項目に1回程度 |
| 3 | 両側存在するが、一方が具体的事実（数字・手口・法律条文）なのに対し、他方が「〜懸念」「〜必要」の一般論で埋まっている |
| 2 | 片側だけ具体的で、反対側が帰属なしスローガン2項目だけ／両側の分量が極端に違う |
| 1 | 片側の主張が全面的に欠落／「反対意見もあるようです」でごまかす／両側が同じことを言っている（対立になってない） |

**ジャッジポイント**: 弱い側の項目を1つずつ見て「資料に基づく具体的な反論が書けているか」を確認。

---

## 軸4: 論点の一致（alignment）— 5点
**両陣営が同じ問いに答えているか（すり替え・噛み合ってないを検出）**

| 点数 | 基準 |
|------|------|
| 5 | 両側ともissueTitleの問いに対して真っ向から賛否を述べている。設問と両セクションの芯が完全一致 |
| 4 | 概ね一致。片側がやや設問の前提をずらして反論しているが、実質的には対立 |
| 3 | 設問と片側の芯が少しズレている（「対応は適切か」に対し片側は「事前管理の問題」を言っている） |
| 2 | 両側が設問の別の側面を言っていて、対立として噛み合っていない |
| 1 | 設問と両セクションが完全に別の話／両側が違うトピックについて議論している |

**ジャッジポイント**: 設問文の主語と動詞を抽出し、両セクションの各項目がそれに対する直接の応答か確認。

---

## 軸5: 帰属の品質（attribution）— 5点
**「誰が言ったか」が適切に示され、かつ冗長でないか**

| 点数 | 基準 |
|------|------|
| 5 | 各主張に媒体名/発言者名の帰属が付き、かつ連続する2文目以降は省略されて簡潔。リードに無駄な帰属なし |
| 4 | 帰属は付いているが、一部が冗長（毎文「〜は報じている」）。ただしリードはすっきり |
| 3 | リードの簡潔さは良いが、両セクションの半分以上の項目に帰属がない（「懸念がある」「必要だ」だけ） |
| 2 | 教科書的一般論（帰属なし）で両セクションが埋まっている／逆にリードにまで帰属だらけでタイパ最悪 |
| 1 | 帰属が全くない／ソース不明の数字や固有名詞が断定で書かれている |

**ジャッジポイント**: weak sideの各項目が「XXは〜と述べている」の形になっているか。リードに「〜と報じている」が1つでもあれば減点。

---

## 軸6: 情報密度（density）— 4点
**無駄な繰り返し・水増しがなく、短い中に情報が詰まっているか**

| 点数 | 基準 |
|------|------|
| 4 | 各セクションが固有の情報を持ち、「いま何が論点か」と両セクション・各社で内容の重複がほぼ無い |
| 3 | 一部で重複があるが、全体としては各セクションが役割を果たしている |
| 2 | 冒頭と両セクションで同じ事実が繰り返される。セクション間で情報の差が無い |
| 1 | 各セクションがほぼ同じ内容の言い換え。1段落読めば全部わかる |

**ジャッジポイント**: 「いま何が論点か」の事実が、両セクションで「根拠」として再掲されていないか。「各社は何を伝えているか」が冒頭の焼き直しになっていないか。

---

## 軸7: 投票選択肢の現実味（voteChoices）— 4点
**「賛成／反対」のラベルが、実際の論争を反映した生きた選択肢か**

| 点数 | 基準 |
|------|------|
| 4 | 選択肢が「〜を支持する／規制すべき」等、具体的な行為や立場を指している。政党名・人物名ではない |
| 3 | 選択肢は具体性があるが、やや抽象寄り（「肯定的に評価する」「懸念する」） |
| 2 | 「賛成」「反対」「わからない」のデフォルトラベル。または政党名が入っている |
| 1 | 投票設問自体が曖昧／選択肢が設問と対応していない／Yes/No以外の第三の立場があるのに2択 |

**ジャッジポイント**: 選択肢だけ見て「何に賛成/反対か」が具体的にイメージできるか。「支持する」だけだとダメ。

---

## 軸8: 読みやすさ（readability）— 4点
**一文の長さ・段落構成・HTML構造としての品質**

| 点数 | 基準 |
|------|------|
| 4 | 一文が適度な長さ（概ね60字以内）。h2が適切に使われ、箇条書きと段落が使い分けられている |
| 3 | 概ね読みやすいが、一部に長文（100字超）がある／h2と本文のバランスがやや崩れている |
| 2 | 一文が詰まりすぎ（1文150字超）／段落だらけで箇条書きが少ない／h2の粒度が不適切 |
| 1 | ほとんどが長文段落。h2が無い／HTML構造が崩れている |

**ジャッジポイント**: 最も長い1文の文字数をカウント。h2が過不足なく使われているか。

---

## 軸9: 証拠の厚みと多様性（evidence）— 4点
**複数の独立したソースからバランスよく構築されているか**

| 点数 | 基準 |
|------|------|
| 4 | 3社以上の独立したメディア/ソースを参照。両論とも複数の出典で裏付けられている |
| 3 | 2社以上のソースあり。ただし一方の論拠が1ソースのみ |
| 2 | 実質1ソースのみ／全体的にソースの種類が少ない |
| 1 | ソースが1つしか見えない／裏付けのない主張が目立つ／全てWikipediaで補っている |

**ジャッジポイント**: 出典セクションのURL数と多様性。両セクションの各項目が複数の独立ソースに分散しているか。

---

## 軸10: 議論の導線設計（debateFlow）— 4点
**記事を読んだあと、ユーザーが「自分ならどう思うか」考えたくなるか**

| 点数 | 基準 |
|------|------|
| 4 | 「まだ分からないこと」が具体的な未確定点を指している。両論が拮抗しており、読者が決断を迫られる構造 |
| 3 | 未確定点はあるが日常会話レベル。議論の接戦感はやや弱い |
| 2 | 未確定点が「今後の動向を注視する必要がある」等のテンプレ。両論の拮抗が薄い |
| 1 | 記事が一方的な結論に傾いている／「議論の余地はない」と読める／未確定点が無い |

**ジャッジポイント**: 記事を読んだ後に「自分はどう思う？」と考えさせられるか。「まだ分からないこと」が逃げのテンプレになっていないか。

---

## 加点枠（最大+3点）

| 条件 | 加点 |
|------|------|
| タイムライン形式の記事で時系列整理が秀逸（日付順・過不足なし） | +1 |
| 「海外ではどう報じられているか」が国内議論を補完する有意義な内容 | +1 |
| 公的統計（e-Stat等）や法律条文を具体的に引用して議論に厚みを出している | +1 |

## 減点枠（最大-5点）

| 条件 | 減点 |
|------|------|
| 冒頭の「いま何が論点か」がleadと内容ズレ（同じ文章になってない） | -2 |
| 両セクションのリスト中に「〜とされる」「〜との見方」等の無主観の帰属不明が3つ以上 | -1/件 |
| 根拠の無い数字・固有名詞（=HELLになるがPublishedまで抜けたもの） | -2/件 |
| issueTitleに使えない文字/記号が含まれている | -1 |
